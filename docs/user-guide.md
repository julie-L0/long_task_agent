# 小柳使用手册

小柳是时间流驱动的行动调度秘书。不是 Todo 清单——它会感知你的状态，在你有空的时候主动推进那些一直拖着的事。

「小柳」的名字来自苏联昆虫学家**亚历山大·柳比歇夫**（Alexander Lyubishchev）。他坚持了56年的时间统计法：每天记录自己把时间花在了哪里，精确到分钟。小柳做的事和他一样——持续记录你的时间流，让长期目标不断裂，积累的数据随时可以复盘：深度工作时长、最高效时段、任务拖延模式，都看得见。

---

## 快速上手

直接用自然语言说话，不需要记命令。几个最常用的：

- **"今天要写周报"** → 建任务，今天见缝插针提醒
- **"周五做 X"** → 建任务，周五出现在待办里
- **"学习30分钟"** → 开始计时，30分钟后来问你做完了吗
- **"面板"** → 看今天和本周的全貌
- **"/new"** → 清空对话历史，重新开始（数据不丢）

## 任务管理

### 创建任务

| 你说 | 小柳做 |
|------|--------|
| 今天要做 X | 建任务，今天截止，见缝插针提醒 |
| 明天/周五/下周做 X | 建任务，那天截止，出现在面板待办 |
| X点提醒我做 Y | 建任务 + 建提醒，到点推送 |
| 学习/跑步/写代码 N 分钟 | 计时闭环（见下方） |
| 每天早上9点跑步 | 建周期任务，完成后自动重建下一个 |

小柳不会因为信息不完整就不存——说了任务名就先存，细节之后补。

每次存入数据，小柳都会以 **✅ 已记录** 开头确认，让你知道这件事已经落地。

### 过期任务处理

截止时间已过的任务会出现在面板「⚠️ 需要处理」区块。你可以：

- **"挪到明天"** / **"下周再做"** → 更新截止日期，任务继续存在
- **"取消掉"** → 确认后标记取消，保留记录

小柳会更新已有任务，不会新建重复条目。

### 查看任务

- **"帮我看看所有任务"** / **"现在有什么任务"** → 按状态分组列出全部
- **"面板"** → 今日待办 + 本周待办 + 打卡 + 长期项目
- **"今天有什么要做的"** → 只看今天

### 修改任务

直接说要改什么，小柳会确认后执行：

- "把X改成明天" → 更新截止日期
- "X的优先级改成高" → 更新优先级
- "X不做了" / "取消X" → 标记取消（保留记录）

### 进度更新

- **"论文写了2000字"** → 更新进度，播报完成率和预计剩余时间
- **"跑步打卡"** → 连续天数 +1
- **"喝了一杯水"** → 今日配额 +1

## 计时闭环

说"学习X分钟"/"跑步30分钟"等明确时长的活动，小柳会：

1. 建任务 + 记录时间线
2. 设 N 分钟后的 check-in 提醒
3. 专注类活动（学习/工作/开会）自动设免打扰

到时间小柳会问"做完了吗？"

- **"完了"** → 关闭计时，恢复可打扰，推下一个待办
- **"中断了"** / **"先不做了"** → 暂停任务，记录中断原因

---

## 打扰控制

### 设置免打扰

- **"开会了"** / **"在写论文"** / **"别烦我"** → 小柳问你几点结束或开完告诉它
- 第一次说某类活动，小柳会问"期间要打扰你吗"，你的选择会存成规则，以后自动执行

### 恢复

- **"好了"** / **"回来了"** → 恢复可打扰，小柳推今日最高优先级待办

### 查询状态

- **"现在可以打扰吗"** / **"我在干嘛"** → 查询当前状态

---

## 每日打卡

适合"每天喝3杯水"/"每天做50个俯卧撑"这类每日配额需求：

- **"每天喝3杯水"** → 建打卡项目，每天重置
- **"喝了一杯"** → 今日 +1，播报进度
- 配额完成后当天不再提醒，第二天自动重置

## 面板

说"面板"/"状态"/"今天怎么样"，小柳输出：

```
📅 5月29日 周四  10:32

📋 今日待办
🔴 写周报
🟡 14:00 和小明对接（60min）
✓ 晨跑

📋 本周待办
5月30日 周五  🔴 提交 PRD
6月1日 周日   🟡 整理笔记（弹性）

💧 今日打卡
喝水  ██░  2/3
学英语  ✓  连续 Day12

📊 长期项目
论文  33%  已 3 天未推进

⚠️ 需要处理
「数据清洗」截止已过
```

优先级：🔴 高 / 🟡 中 / ⚪ 低

## 上下文管理

### /new 或"切线"

清空对话历史，减轻 Agent 负担。**数据全部保留**，新对话里小柳仍然知道你所有的任务和项目。

小柳会先检查对话里有没有漏存的内容，补完后再清。

### "整理一下" / "归档"

逐个确认哪些任务完成了、哪些不要了，然后归档。比切线更彻底，适合定期清理。

## 自定义规则

### 定时提醒

- **"每天晚上10点提醒我喝水"** → 建每日规则
- **"每周一早上提醒我看任务列表"** → 建每周规则
- 持续提醒直到你回复：加"直到我回复"/"提醒到我确认"

回应持续提醒说"好了"/"知道了"，小柳停止当天的重复提醒。

### 活动规则

第一次说某类活动（开会、写论文、通勤），小柳会问期间要不要打扰，你的选择自动存成规则，以后不再问。

### 沟通偏好

第一次使用时小柳会问你希望怎么沟通（"简短直接"/"每次说清楚原因"等），存成偏好后每次对话都生效。

## 主动推进逻辑

小柳会在以下情况主动找你：

| 触发条件 | 小柳做什么 |
|---------|-----------|
| 你空闲 + 本周有 deferrable 任务 | 推优先级最高的一个 |
| 每日配额未完成 + 你空闲 | 提醒还差几次 |
| 任务/项目 24 小时未推进 | 问是否有空处理（4小时内不重复） |
| 今天截止 + 晚上8点后 | 问现在做还是挪到明天 |
| 60 分钟没消息 | 问在干嘛，记录时间线 |

你忙的时候（dnd 状态）所有主动推送暂停。

## 数据分析

小柳记录的时间线数据可以分析：

```bash
npm run analyze           # 终端报告
npm run analyze:export    # 同时导出 CSV
```

分析维度：每日活动分布、深度工作趋势、任务拖延、项目进度速率、最高效时段、打断原因。

## 常见问题

**Q: 说了任务但面板里没有**
A: 说"/new"让小柳检查有没有漏存，或者直接问"帮我看看所有任务"。

**Q: 提醒一直在响停不下来**
A: 回复提醒内容（"好了"/"知道了"）即可停止当天的重复提醒。

**Q: 想改沟通偏好**
A: 直接说"以后说话简短点"或"帮我更新一下沟通偏好"。

**Q: 想看历史数据**
A: 运行 `npm run analyze`，或直接查飞书多维表格（如果用了 bitable 存储）。
